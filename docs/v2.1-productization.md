# Canonical CAT v2.1 Productization

v2.1 adds the first product-facing path on top of the v2.0 reference core: a Codex-style adapter, a runnable CLI demo, a sample trace, persisted state, and package metadata.

## What changed

- `adapters/base.py` defines the adapter protocol from the public contract.
- `adapters/codex_adapter.py` converts Codex-like tool/action JSON records into canonical `StepEvent` objects.
- `scripts/cat_demo.py` runs the adapter plus `run_cat_cycle` over a JSONL trace.
- `examples/codex_trace_sample.jsonl` provides a small reproducible trace.
- `cat_core/storage.py` persists and reloads `CATState` as JSON.
- `pyproject.toml` makes the repository installable and exposes the `cat-demo` command.

## Run it

```bash
python scripts/cat_demo.py \
  --input examples/codex_trace_sample.jsonl \
  --output runs/demo.cat.jsonl \
  --state runs/demo.state.json
```

or after installation:

```bash
pip install -e .
cat-demo \
  --input examples/codex_trace_sample.jsonl \
  --output runs/demo.cat.jsonl \
  --state runs/demo.state.json
```

## Why this matters

The v2.0 core proved the CAT loop. v2.1 proves the product journey: external runtime events can enter through an adapter, flow through the canonical cycle, produce logs, and persist memory for future turns.
